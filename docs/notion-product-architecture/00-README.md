# GTM OS Product Architecture Pack

Snapshot date: February 8, 2026

This folder is a product-first documentation pack for Notion.
It explains what the app is, why each part exists, and what to iterate next.

## How to use this pack

1. Start with `01-Product-Intent.md`.
2. Read `02-Experience-Architecture.md` to understand the full system flow.
3. Go deeper by area:
   - `03-Roadmap-and-Planning-Design.md`
   - `04-Execution-Design.md`
   - `05-Knowledge-System-Design.md`
   - `06-AI-Companion-Design.md`
   - `07-Operating-Model-and-Governance.md`
4. Use `08-Current-State-to-Target-Iteration-Backlog.md` to prioritize next cycles.

## What this pack is

- Product strategy and design intent
- User workflows and experience architecture
- Current state framing and iteration direction

## What this pack is not

- Deep technical architecture
- Implementation details
- Code-level specifications
